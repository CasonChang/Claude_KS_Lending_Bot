## 六、天期邏輯待決議（本週檢討定奪時機）

上週提出的改法（見 `reviews/README.md` 未決議事項），本週資料再次確認：

**問題**：`config.yaml` 的 `periods` 門檻 120天=18%、30天=12%，
- fUSD 本週頂檔(×1.45)最高才 ~15%，**仍踩不到 120天門檻**
- fUST 30天那 9 筆成交均 14.41%、放滿率 0.3%（被 refinance 秒還）

**建議（保守版，可先上）**：

```yaml
# 現況
periods:
  - { apy: 18, days: 120 }
  - { apy: 12, days: 30 }
  - { apy: 8,  days: 7 }
  - { apy: 0,  days: 2 }

# 建議保守版（先改 30天門檻，讓市場價基/中檔開始掛 30天）
periods:
  - { apy: 16, days: 120 }   # 小降，偶發大 spike 時才觸發
  - { apy: 9,  days: 30 }    # 主改：錨點×1.0(9%)就掛 30天，而非等 12%
  - { apy: 6,  days: 7 }     # 稍降
  - { apy: 0,  days: 2 }
```

效果：錨點 9% 時，基檔(×1.0)=9% → 30天（之前是 2天）。**核心改動**是讓近市場價的 50% 資金開始鎖 30天，停止每 2 天落地閒置一次。

⚠️ **取捨**：若之後利率大漲，30天的單會卡在舊價。
**驗證指標**：31–120天成交占比、長單放滿%、fUSD 實收淨年化（目標 >8%）。
