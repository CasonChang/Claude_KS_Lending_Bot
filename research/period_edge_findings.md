# fUSD 績效落後診斷：是 (a) 沒掛長單 / (b) 掛了不成交 / (c) 成交被早還？

**日期**：2026-06-24（本機唯讀分析，未動策略、未下/撤單）
**腳本**：`research/period_edge_analysis.py`（可重跑；市場端公開 API 易被限流，必要時分頁加大）

## TL;DR

**主因是 (a)——我們系統性地幾乎不掛 30/120 天長單，而市場的大錢與高利率幾乎都在長天期。**
(b) 可排除（市場長單需求很大）。(c) 真實存在但是**次要、且部分被誤解**：我們「越長天期反而越快被早還」，
是因為我們的長單都掛在 spike 高利率、利率一回落借款人就還掉——**不是長天期本身的問題，是「掛太高」**。

對手能穩定 ~15%，edge 不是「spike 時鎖高利」，而是**長期維持高占用率、用接近市場價的長天期單把錢黏住領很久**。

## 診斷

- **(a) 沒掛夠長 → 成立，且是主因。** 我們的放貸中 + 近 500 筆已結束，**31–120 天合計 0 筆**，
  從來沒成交過一筆長單；而市場 60% 量、80% 高利率量都在這。我們大部分資金壓在 2 天（甚至 @4.3%，比市場 6.5% 還低）。
  **根因在 `config.yaml` 的天期門檻**：120 天要該檔年化 ≥18%、30 天要 ≥12%。我們頂檔 = 錨點×1.45，
  fUSD 錨點長期在 9–13%，頂檔頂多 ~13–18%，**幾乎踩不到 18% → 120 天門檻形同虛設**；
  市場明明 11% 就有長單可掛，我們卻在等 18%。

- **(b) 掛了長單不成交 → 排除。** 市場長單需求充沛（60% 量），不是沒人借；是我們根本沒去掛。

- **(c) 成交被早還 → 成立但次要、且被誤解。** 我們「越長天期早還越誇張」（8–30 天放滿 0%、100% 早還），
  表面像長單沒用；**真因是我們的長單都掛在 spike 高利（8–30 天成交均 12.9%），利率一回落借款人立刻還掉再借便宜的**。
  也就是「掛太高才被早還」，不是「長天期沒用」。對手用**接近市場價**的長單，借款人沒有誘因 refinance，就黏住領很久。
  （另一個偏誤：已結束樣本天生偏向「還得快」的單，真正黏住的還在「放貸中」沒進這張表。）

## 建議調整方向（先不改，等決定）

**方向 1（主修，對應 a）：把 `config.yaml` 的天期門檻往下調，讓長單在「市場價」就觸發，而不是等 spike。**

```yaml
# 現況（120 天要 18%，幾乎踩不到）
periods:
  - { apy: 18, days: 120 }
  - { apy: 12, days: 30 }
  - { apy: 8,  days: 7 }
  - { apy: 0,  days: 2 }

# 建議（貼合市場：長單 11% 就出沒）
periods:
  - { apy: 11, days: 120 }
  - { apy: 8,  days: 30 }
  - { apy: 5,  days: 7 }
  - { apy: 0,  days: 2 }
```

效果：錨點 9% 時，頂檔(×1.45)=13% → 120 天、中檔(×1.15)=10.4% → 30 天。開始真正吃到長天期市場。
⚠️ 取捨：把較多資金鎖長天期會降低靈活度、且若之後利率大漲手上長單卡在舊價。
建議**搭配方向 2 控制比例**，不要 base 檔也全鎖長。

**方向 2（對應 c，避免「掛太高被早還」）：長天期檔不要追 spike 溢價，掛在貼近市場的價位讓借款人黏住。**
具體可在 `strategy.py` 的 `build_ladder`／天期決策裡，讓**長天期檔的利率倍率收斂**（例如長單檔 mult 靠近 1.0），
用「市場價長單」換「被長期持有、複利累積」，而不是「高價長單、秒被 refinance」。
（或更簡單：spike_ladder 維持追高領短期財，但**正常 ladder 的長天期檔壓低 mult**。）

**驗證指標**：改完後追蹤「31–120 天的成交量占比」「長單平均放滿%」「整體 fUSD 實收淨年化」三者是否上升。
